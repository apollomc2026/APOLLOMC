# Startup Minimal — Style Specification
**Version:** 1
**Industry:** Startup

## Typography
- Heading font: SF Pro Display (fallback: -apple-system, Segoe UI)
- Body font: SF Pro Text (fallback: -apple-system, Segoe UI)
- Heading sizes: H1 32pt medium weight, H2 24pt medium weight, H3 18pt medium weight, H4 14pt semibold
- Body size: 14pt for slides, 11pt for narrative documents
- Line height: 1.65
- Stat callout size: 48pt SF Pro Display light
- Caption: 10pt SF Pro Text light, tracking +0.5pt
- Label text: 9pt SF Pro Text medium, uppercase, tracking +1pt

## Color palette
- Primary: #1A1A2E — use for headings, primary text, and key metric numbers
- Secondary: #16213E — use for secondary headings and dark section backgrounds
- Accent: #0F3460 — use for links, interactive elements, and subtle emphasis
- Highlight: #E94560 — use extremely sparingly — one element per page maximum for critical emphasis
- Body text: #1A1A2E
- Secondary text: #6B7280
- Background: #FFFFFF
- Alternate section background: #FAFBFC
- Card background: #F9FAFB with 1pt #E5E7EB border
- Chart palette: #1A1A2E, #374151, #6B7280, #9CA3AF, #E94560 (accent only)

## Layout rules
- Page size: 16:9 widescreen for presentations; 8.5in x 11in for documents
- Margins: 0.8in all sides for slides; 1in for documents
- Section spacing: 40pt before H1, 28pt before H2, 16pt before H3
- Paragraph spacing: 12pt after each paragraph
- Content per slide: one idea, maximum 3 lines of text
- Column structure: centered single column preferred; two-column for data comparisons only
- Slide numbers: bottom right, 9pt, #9CA3AF
- No headers, no footers on slides — absolute minimal chrome
- Section dividers: extra whitespace (60pt) or a new slide with single centered heading
- Generous negative space — at least 30% of every slide should be empty

## Formatting conventions
- Stat callouts: 48pt light weight, centered, with label above in 9pt uppercase tracking
- No boxes, no borders, no background fills on callouts — numbers stand alone on white space
- Charts: thin-line only, no fills, no gridlines, axis labels in 9pt #9CA3AF
- Sparklines preferred over full charts where trend is the message
- Tables: no borders except light gray bottom rule per row, no header background fill
- Lists: no bullet characters — use indentation and spacing as the only hierarchy indicator
- Icons: thin-line stroke style only (1.5pt), monochrome #1A1A2E
- Team: name and title only, no bios, no headshots — let the names speak
- Product images: clean device mockups with generous surrounding whitespace
- No gradients, no shadows, no rounded corners on containers
- The single red accent (#E94560) reserved for: the revenue number, the ask amount, or one key CTA

## Tone and voice
- Restrained, intelligent, and quietly confident
- Write as a second-time founder who does not need to shout
- Let the product and numbers speak — minimal commentary
- Short, precise sentences. No filler words. No intensifiers.
- Do not use "revolutionary," "disruptive," "game-changing" or any similar buzzwords
- State what you built, what it does, and who pays for it
- Numbers presented cleanly without exclamation: "$8.2M ARR" not "$8.2M ARR!"
- Problem stated in one sentence. Solution in one sentence. The rest is evidence.
- "We built X. It does Y. Z customers pay for it. We need $A to reach B."
- Sophistication through brevity — every word earns its place

## What this style MUST produce
- Refined, Apple-aesthetic pitch materials that signal design sophistication
- Maximum whitespace — at least 30% negative space on every slide
- No decorative elements: no boxes, no colored backgrounds, no gradients
- Thin-line charts and sparklines only for data visualization
- One red accent element per document for the single most important number
- Ultra-concise text: maximum 3 lines per slide, maximum 8 words per line
- Clean typography hierarchy as the primary design tool
- Team presented as names and titles only
- Product shown in clean device mockups
- Slide count: per deliverable type specification (typically 12-20 slides)
- All sections in the order specified by the deliverable module
