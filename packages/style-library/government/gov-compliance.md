# Government Compliance — Style Specification
**Version:** 1
**Industry:** Government

## Typography
- Heading font: Times New Roman
- Body font: Times New Roman
- Heading sizes: H1 16pt bold, H2 14pt bold, H3 12pt bold, H4 12pt bold underline
- Body size: 12pt
- Line height: 1.5
- Table text: 10pt
- Regulation citation text: 11pt italic
- Footnote size: 10pt

## Color palette
- Primary: #2C2C2C — use for headings, section rules, and emphasis
- Secondary: #555555 — use for subheadings and secondary labels
- Alert: #8B0000 — use for non-compliance findings, critical gaps, and corrective action flags
- Pass indicator: #2E5E3E — use for compliant status indicators only
- Body text: #2C2C2C
- Background: #FFFFFF
- Finding box background: #FFF5F5 (non-compliant) or #F5FFF5 (compliant)
- Table header background: #E8E8E8 with #2C2C2C text
- Table borders: #CCCCCC, 1pt solid

## Layout rules
- Page size: 8.5in x 11in (US Letter)
- Margins: 1in all sides
- Section spacing: 18pt before H1, 14pt before H2, 8pt before H3
- Paragraph spacing: 6pt after each paragraph
- Column structure: single column exclusively
- Page numbers: bottom center, "Page X of Y", 10pt
- Header: "COMPLIANCE ASSESSMENT — [REGULATION NAME]" centered, 10pt bold, with #2C2C2C rule below
- Footer: classification level centered (e.g., "FOR OFFICIAL USE ONLY"), date right-aligned, 9pt
- Document control: version number, date, author, and review status on title page

## Formatting conventions
- Regulatory citations formatted precisely: "48 CFR 52.204-21(b)(1)(ii)" — no abbreviation of citation format
- Finding blocks: bordered boxes with status indicator (COMPLIANT in green, NON-COMPLIANT in red, PARTIAL in amber)
- Each finding includes: requirement reference, finding description, evidence reviewed, risk level, corrective action
- Compliance matrix: required for all reports — table mapping each regulation clause to compliance status
- Risk ratings: Critical / High / Medium / Low with corresponding #8B0000 / #B44A00 / #B8860B / #2E5E3E
- Evidence references: numbered (E-1, E-2, E-3) with full descriptions in an evidence appendix
- Corrective action plans: numbered (CAP-1, CAP-2) with owner, deadline, and verification method
- Cross-reference table of acronyms and definitions required
- Version control table on page 2 showing all document revisions

## Tone and voice
- Regulatory, precise, and impartial
- Write as an independent assessor — never advocate for or against the assessed entity
- State findings as facts: "The organization does not maintain" not "The organization fails to maintain"
- Use regulatory language precisely — do not paraphrase requirements, quote them
- Passive voice preferred for findings: "It was observed that" or "Evidence indicates that"
- No subjective qualifiers: avoid "significant," "substantial," "adequate" without defining the threshold
- Severity language must match the risk framework: "Critical" means specific operational impact, not emphasis
- Recommendations phrased as requirements: "The organization must implement" or "Corrective action is required"
- Timeline language must be specific: "within 30 calendar days" not "promptly"

## What this style MUST produce
- Audit-grade compliance documents suitable for regulatory submission
- Compliance matrix mapping every applicable requirement to a finding
- Finding blocks with standardized format: reference, description, evidence, risk, corrective action
- Risk-rated findings using the four-tier framework (Critical/High/Medium/Low)
- Evidence appendix with numbered references
- Corrective action plan table with owners and deadlines
- Version control table
- Acronym and definitions glossary
- Page count: per deliverable type specification
- All sections in the order specified by the deliverable module
