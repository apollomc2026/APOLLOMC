# Government Proposal — Style Specification
**Version:** 1
**Industry:** Government

## Typography
- Heading font: Calibri
- Body font: Calibri
- Heading sizes: H1 18pt bold, H2 15pt bold, H3 12pt bold, H4 11pt bold italic
- Body size: 11pt
- Line height: 1.4
- Table text: 10pt
- Callout text: 11pt bold
- Footer/header text: 9pt

## Color palette
- Primary: #003366 — use for headings, section dividers, compliance header bars, and table headers
- Secondary: #4A7FB5 — use for subheading accents, secondary table headers, and figure borders
- Accent: #B8860B — use sparingly for win theme callouts and discriminator highlights only
- Body text: #1A1A1A
- Background: #FFFFFF
- Compliance callout background: #EBF0F7
- Table header background: #003366 with white text
- Table alternating rows: #F5F7FA
- Win theme box background: #FFFDE8 with #B8860B left border

## Layout rules
- Page size: 8.5in x 11in (US Letter)
- Margins: 1in all sides
- Section spacing: 18pt before H1, 14pt before H2, 8pt before H3
- Paragraph spacing: 6pt after each paragraph
- Column structure: single column primary; two-column for organizational charts and team layouts
- Page numbers: bottom center, "Page X of Y" format, 9pt
- Header: solicitation number left, volume/section title center, company name right, 9pt, #003366 underline rule
- Footer: "PROPRIETARY — USE OR DISCLOSURE RESTRICTED" centered, 8pt bold #888888
- Compliance header bars: navy (#003366) bars with white text labeling each RFP section requirement

## Formatting conventions
- Section numbering mirrors solicitation structure (L, M, or 1.0, 2.0 as RFP dictates)
- Compliance callout boxes: #EBF0F7 background with navy left border, containing the RFP requirement text verbatim followed by "Our Approach:" response
- Win theme boxes: gold-bordered (#B8860B), placed at the start of each major section, summarizing the key discriminator
- Past performance references: structured table format (contract name, agency, value, period, relevance)
- Organizational charts: boxes with navy headers, clear reporting lines
- Action verbs in proposal text: "We will deliver," "Our team provides," "This approach ensures"
- Feature-benefit tables: two-column layout mapping features to evaluator benefits
- Graphics numbered sequentially with descriptive captions below
- Resumes in standardized format: name, role, years of experience, clearance, relevant projects

## Tone and voice
- Professional, confident, and evaluator-focused
- Write directly to the evaluation criteria — every section should score points
- Use "we" for the proposing entity and "you" or agency name for the customer
- Confident without arrogance: "Our proven approach ensures" not "We are the best"
- Quantify every claim: specific numbers, dates, dollar amounts, percentages
- Ghost the competition subtly: "Unlike approaches that require..." without naming competitors
- Demonstrate understanding before proposing solutions: restate the problem, then solve it
- Use benefit statements: not just "we will do X" but "this delivers Y benefit to the agency"
- Compliance language: "The offeror understands and will comply with" for mandatory requirements

## What this style MUST produce
- Proposal-ready documents that align precisely with solicitation evaluation criteria
- Compliance headers or callout boxes mapping every response to its RFP requirement
- Win theme boxes at the start of each major section
- Past performance in structured tabular format
- Feature-benefit linkage visible throughout
- Proprietary notice on every page footer
- Solicitation number in the header of every page
- Professional organizational charts where team structure is relevant
- Page count: per deliverable type specification and solicitation limits
- All sections in the order specified by the deliverable module or solicitation
